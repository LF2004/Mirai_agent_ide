# Mirai Agent IDE 需求整理

## 已完成

- Agent 基础对话链路已打通，支持本地会话持久化与重启恢复。
- Agent 支持 `agent` / `ask` / `plan` / `debug` 模式切换。
- 模型配置已支持本地增删改查，并写入 SQLite + `agent-config.json` 兜底。
- 已接入 OpenAI 兼容接口与 `Responses API`。
- 已补充代理自动识别，优先读取系统代理与环境代理。
- Agent 面板已重做为更紧凑的 Cursor 风格布局，包含会话 tabs、状态栏、消息流、composer。
- 工具默认路径空值导致的报错已修复一批。

## 还差什么

- 继续完善 Cursor 风格的 Agent UI，包括顶部会话栏、消息气泡、tool call 细节、请求 UUID 复制入口。
- 补齐更完整的多任务编排能力，真正打通 `plan -> task -> execute -> verify`。
- 增强上下文管理与长对话压缩策略。
- 把 aborted、工具报错、请求失败做成更清晰的日志与 UI 提示。
- 增加本地历史对话列表、搜索、删除、导出等管理能力。

## 后续建议

- 先把普通 Agent 能力做到稳定可用，再继续做多任务与 plan 工作流。
- 接着统一整个 Agent 面板的视觉语言，让它更接近 Cursor。
- 最后再补自动化测试和诊断面板，方便排查模型与工具问题。
